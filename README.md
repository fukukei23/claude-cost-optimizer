# claude-cost-optimizer

## 概要

Claude Code CLI のバックエンドを Z.AI（GLM-5.1）や MiniMax に切り替え、API利用料金を大幅に削減するためのスクリプトと設定テンプレートを公開しています。

主に次のことができます。

- Claude Code CLI の処理を GLM-5.1 または MiniMax-2.7 にルーティングする設定の雛形を提供する
- GLM-5.1 へのリクエストが失敗した場合、MiniMax-2.7 へ自動的にフォールバックする仕組みを提供する
- あらかじめ用意されたテンプレートをもとに、自身の環境に合わせた設定を構築する

## コスト比較

2026年4月時点のAPI従量課金に基づく1Mトークンあたりの概算です。

| モデル | 1Mトークン合計 | Sonnet 4.6比 |
|---|---|---|
| Claude Sonnet 4.6 | 8.00 | 100%（基準） |
| GLM-5.1 | .75 | 約26%（74%削減） |
| MiniMax-2.7 | .50 | 約8%（92%削減） |

GLM-5.1 を主系に据え、エラー時に MiniMax-2.7 へ逃がす構成をとることで、コストを抑えつつ可用性を維持できます。

## 収録ファイル

| ファイル | 内容 |
|---|---|
|  | GLM-5.1 へのリクエスト失敗時に MiniMax-2.7 へ自動フォールバックするスクリプト |
|  | Claude Code CLI 向けの設定テンプレート。値はサニタイズ済み |
|  | フォールバック動作の条件や閾値を定義する設定ファイルテンプレート |
|  | フォールバック構成の設計思想と動作の詳細説明 |

## 導入について

本リポジトリは、APIの認証情報管理やCLIの設定ファイル編集、Pythonスクリプトの実行環境構築に一定の知識を前提としています。

各テンプレートの値をご自身の環境に合わせて書き換える必要があり、設定によってはAPIキーの取得や権限の調整も発生します。そのままでは動作しないため、ご自身の環境に適用する際は適宜調整してください。
