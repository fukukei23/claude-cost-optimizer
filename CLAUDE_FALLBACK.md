# Claude コスト最適化フォールバックスクリプト

## 概要

本リポジトリでは、Claude Code CLI 利用時の API コストを大幅に削減するためのフォールバック機構を公開しています。

このスクリプトは、サードパーティーの互換 API プロバイダーを経由して Claude Code CLI を実行し、通信エラーやタイムアウトが発生した際に別のプロバイダーへ自動的にリクエストを再送します。実際の開発業務にて運用しており、安定した動作を確認しています。

## 仕組み

公開しているスクリプトは、主に以下のコンポーネントで構成されています。

- **エラー分類モジュール** - CLI の終了コードや標準エラー出力を解析し、エラーを再試行可能なものと不可能なものに分類します。
- **環境変数生成モジュール** - フォールバック先のプロバイダー向けに、環境変数のセットを動的に生成します。
- **プロセス実行モジュール** - 任意の環境変数を適用した状態で Claude CLI を実行し、プロセスの終了を待機します。

基本的な動作の流れは以下の通りです。

1. 通常時は、Claude コマンドを優先プロバイダーへ送信します。
2. タイムアウトまたは再試行可能なエラーが発生した場合、同じコマンドをセカンダリープロバイダーへ再送します。
3. 実行結果に関わらず、利用したプロバイダー名、レイテンシ（ミリ秒）、終了コードをログファイルに記録します。

プロバイダーの切り替え先やタイムアウト時間などの設定は、ユーザーディレクトリ内の設定ファイルにて一元管理されます。

## コスト比較

本仕組みを利用した場合の、プロバイダーごとの相対コスト（実測値）は以下の通りです。Anthropic 正規 API を基準とした際の割合を示しています。

| プロバイダー | 相対コスト |
|---|---|
| Anthropic Sonnet | 100%（基準） |
| Z.AI GLM-5.1 | 約26%（74%削減） |
| MiniMax | 約8%（92%削減） |

GLM-5.1 を主系に据え、エラー時に MiniMax-2.7 へ逃がす構成をとることで、コストを抑えつつ可用性を維持できます。

## 導入の流れ

導入にあたっては、リポジトリ内の設定ファイル例を参考に、ご自身の環境にてプロバイダー別の認証情報とエンドポイントを設定していただく必要があります。

公開しているスクリプトと設定例を組み合わせることで、ローカル環境にてフォールバック機構を直ちに有効化できます。
